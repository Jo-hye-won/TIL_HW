## 너비 우선 탐색 
> 탐색 시작점의 인접한 정점들을 먼저 모두 차례로 방문한 후에, 
> 방문했던 정점을 시작점으로 하여 다시 인접한 정점들을 차례로 방문하는 방식
> 인접한 정점들에 대한 탐색을 한 후, 차례로 다시 너비우선탐색을 진행해야 하므로, 
> 선입선출 형태의 자료구조인 큐를 활용함

1. 초기 상태
 - visited 배열 초기화
 - Q 생성
 - 시작점 enqueue
 + 시작점에 대해서 visited 표시

2. A점부터 시작 
 - dequeue : A
 - A 방문한 것으로 표시
 - A의 인점점 enqueue
   + enqueue 하면서 들렸다는 표시

3. 탐색 진행
 - dequeue : B
 - B 방문한 것으로 표시
 - B의 인점점 enqueue
   + enqueue 하면서 들렸다는 표시
   
 - dequeue : C
 - C 방문한 것으로 표시
 - C의 인점점 enqueue
   + enqueue 하면서 들렸다는 표시
